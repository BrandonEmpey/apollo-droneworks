# Refresh Hero with Aerial Drone Images + Global Image Rule

## What & Why
The Hero section currently uses three Unsplash photos. Replace them with three brand-new aerial drone-perspective images (no drone visible in frame), and lock in a site-wide rule: every image generated for this site — including AI-generated blog cover images and any future imagery — must be a top-down or oblique aerial view as if shot by a drone, with no drone, props, or watermarks visible.

## Done looks like
- The Hero carousel shows three brand-new high-resolution aerial images (16:9) of Southern Utah / commercial drone subject matter, none of which contain a visible drone.
- Returning visitors see the new images automatically (the localStorage version key is bumped so the previously cached defaults are replaced).
- The blog generator (`server/ai/blog-generator.ts`) instructs Grok to write image prompts that always describe an aerial drone-view scene with no drone visible, and adds the same rule as a hardcoded suffix to the prompt sent to grok-2-image — so even if Grok ignores the system prompt, the image model still receives the constraint.
- `replit.md` documents the global imagery rule under the "User Preferences" section so future agents follow it without being told.

## Out of scope
- Redesigning the Hero layout, animation, or typography.
- Replacing imagery on other pages (about, services, industry, blog index) beyond what is generated by the blog cron going forward. Existing images on those pages stay as-is until separately requested.
- Generating videos for the Hero — only still images are in scope here.

## Steps
1. **Generate the three Hero images** — Create three 16:9 aerial drone-perspective stills (no drone in frame), saved into `attached_assets/generated_images/`. Suggested subjects to keep variety: (a) a Southern Utah red rock canyon at golden hour, (b) a luxury estate / property with surrounding land from above, (c) a working construction or commercial site from above. Each prompt must explicitly include "aerial drone-view perspective, no drone or aircraft visible in frame" and a negative prompt of "drone, quadcopter, propellers, watermark, text".
2. **Wire the new images into the Hero carousel** — Update `HERO_OPTIONS` in `client/src/components/hero-section.tsx` so `image-1`, `image-2`, `image-3` reference the three new local assets via the `@assets/...` import alias instead of Unsplash URLs. Update each item's `title` to match the new subject. Bump `HERO_DEFAULTS_VERSION` to `southern-utah-v2` so cached visitors get the new defaults on next load.
3. **Enforce the global aerial rule in AI image generation** — In `server/ai/blog-generator.ts`, (a) extend the system prompt's `imagePrompt` field description so Grok is required to produce an aerial drone-view scene with no drone visible, and (b) append a fixed suffix (`" — aerial drone-view perspective, photorealistic, no drone or aircraft visible in frame, no watermark, no text"`) to the prompt sent to `grok.images.generate(...)` as a safety net.
4. **Document the rule** — Add a short "Imagery guidelines" line to the `## User Preferences` section of `replit.md`: all site imagery, AI-generated or otherwise, must be aerial drone-perspective with no drone visible.
5. **Verify** — Restart the Start application workflow, open the home page, and confirm the Hero cycles through the three new aerial images with no drones visible. Run `npx vitest run` and `npx eslint client/src/ && npx tsx scripts/check-hub-features.ts` to confirm nothing regressed.
